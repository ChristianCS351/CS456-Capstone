# CS456-Capstone - Pantry Pilot

# Overview

What is Pantry Pilot?
Have you ever been in the grocery store questioning what you need to get? How about when you go home and realize you forgot something you really needed to buy? We have, and that is why we wanted to create Pantry Pilot.

Pantry Pilot is our answer to grocery shopping. With simple controls and an easy to learn system, anyone can use Pantry Pilot for their grocery needs. You begin by creating a username and password. The password is securely hashed using PHP's implemented "password hash" function. 

After you create your account, you go in and begin building your personalized pantry! With our website, you are able to add items with their name, quantity, expiration date and location. We also have implemented a barcode feature that will open your camera, scan the barcode, and automatically input certain information. We also have created a shopping list feature that allows you to input a name as well as quantity that you want to buy. We have added a save and print function to your shopping list so that you can easily access your shopping list on the go. 

# How Does the Website Work?

To create Pantry Pilot, we used a variety of resources. We used PHP as our primary coding language, then we used SQL for implemnting databases, AMPPS to develop our environment and create the physical website, and javascript for visual elements. After our initial creation of Pantry Pilot, our Professor introduced us to Google's new Antigravity system. Antigravity is a fully-featured IDE that implements AI technology. We used Antigravity to see what it thought about our website and what it thought we could improve upon, and it helped us a lot with learning a better, more efficient visual design that is much more appealing. 

# What to You Need to Install?

If you want to work on improving Pantry Pilot, it is easy to set everything up! Starting out, you need to download PHP and Javascript (we used Visual Studio Code for ours), as well as AMPPS. After this, you link the Pantry Pilot code into the AMPPS and begin working! With our databases, we used the SQL inside of AMPPS, so you will have to create your own tables.
